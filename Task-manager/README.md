# Task Manager

REST API сервис для управления задачами с отправкой событий в Kafka.

## Технологии

- **Java 21**
- **Spring Boot**
- **Spring Data JPA**
- **Spring Kafka**
- **H2 Database (для проверки)**
- **Lombok**
- **Maven**
- **Docker**

## Функциональность

- Получение задач с пагинацией
- Получение задачи по ID
- Создание задачи
- Назначение исполнителя задаче
- Смена статуса задачи
- Отправка событий в Kafka
- Авторизацию и онбординг пользователей оставить за рамками тестового задания

## Модели данных

### Пользователь (User)

| Поле  | Тип    | Описание                 |
|-------|--------|--------------------------|
| id    | Long   | Уникальный идентификатор |
| name  | String | Имя пользователя         |
| email | String | Почта (уникальное поле)  |

### Задача (Task)

| Поле        | Тип        | Описание                                                             |
|-------------|------------|----------------------------------------------------------------------|
| id          | Long       | Уникальный идентификатор                                             |
| name        | String     | Наименование задачи                                                  |
| description | String     | Описание задачи                                                      |
| status      | TaskStatus | Статус задачи (CREATED, ASSIGNED, IN_PROGRESS, COMPLETED, CANCELLED) |
| assignee    | User       | Исполнитель (связь с пользователем)                                  |

### Связи между моделями

- **Task** → **User**: Many-to-One (много задач может быть назначено одному пользователю)
- Поле `assignee` в задаче может быть `null` (исполнитель не назначен)

## Статусы задач

| Статус      | Описание                                |
|-------------|-----------------------------------------|
| CREATED     | Задача создана, исполнитель не назначен |
| ASSIGNED    | Исполнитель назначен                    |
| IN_PROGRESS | Задача в работе                         |
| COMPLETED   | Задача завершена                        |
| CANCELLED   | Задача отменена                         |

### Правила смены статусов

- `CREATED` → `ASSIGNED` → `IN_PROGRESS` → `COMPLETED`
- `ASSIGNED` → `CANCELLED`
- `IN_PROGRESS` → `CANCELLED`
- Запрещён переход из `CREATED` сразу в `COMPLETED` или `CANCELLED`
- Нельзя изменить статус завершённой или отменённой задачи
